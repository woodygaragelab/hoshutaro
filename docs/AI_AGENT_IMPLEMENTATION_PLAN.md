# AIエージェント星取表編集機能 実装プラン

## Context

HOSHUTAROは現在フロントエンドのみ(React 19 + MUI + Zustand)の保全管理アプリで、AIアシスタントはMockAIService（パターンマッチ）のみ。本プランでは、Python FastAPIバックエンドを新設し、LangGraphベースのAIエージェントによる星取表データの自律編集機能を実装する。LoopOneプロジェクトのLLMアダプター・LangGraphワークフロー・設定UIパターンを参考にする。

## 機能別プロセス設計

### 機能1: Excel取込み → 星取表データ変換

#### Phase 1: 構造解析 (LLMによるヘッダー・凡例・レイアウト推定)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ INPUT                                                                       │
│                                                                             │
│  User: Excelファイル (.xlsx/.csv) をドラッグ&ドロップ                         │
│  ↓                                                                          │
│  Backend: openpyxlで全シート読込み (バイナリ受信)                              │
│    → セル結合情報 (merged_cells) も取得                                       │
│    → 全シート名リスト取得                                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│ STEP 1: ヘッダー行の推定 (LLM)                                               │
│                                                                             │
│  問題: ヘッダーが1行目とは限らない (タイトル行、空行、説明行が先行する)          │
│                                                                             │
│  処理:                                                                       │
│    1. 各シートの先頭20行を抽出 (セル値 + セル書式 + 結合情報)                  │
│    2. LLMに構造解析を依頼:                                                    │
│       ┌──────────────────────────────────────────────────┐                  │
│       │ System: "Excelシートのヘッダー行を特定してください。               │  │
│       │  以下の手がかりを使用:                                             │  │
│       │  - 連続した非空セルの行 (列名パターン)                             │  │
│       │  - 太字/背景色などの書式がある行                                   │  │
│       │  - その直下にデータが続く行                                        │  │
│       │  - タイトル行、説明行、空行はヘッダーではない"                      │  │
│       │                                                                    │  │
│       │ User: "以下は先頭20行のデータです:                                 │  │
│       │  Row1: ['2024年度 保全計画表', None, None, ...]                    │  │
│       │  Row2: ['作成日: 2024/4/1', None, None, ...]                      │  │
│       │  Row3: [None, None, None, ...]                                    │  │
│       │  Row4: ['設備番号', '設備名', '点検項目', '4月', '5月', ...]       │  │
│       │  Row5: ['P-101', 'ポンプA', '日常点検', '○', '○', ...]           │  │
│       │  ...                                                               │  │
│       │  セル結合: A1:F1 (タイトル結合)                                     │  │
│       │  書式: Row4は太字+背景グレー"                                      │  │
│       │                                                                    │  │
│       │ → LLM出力:                                                        │  │
│       │   { header_row: 4,                                                 │  │
│       │     title_rows: [1, 2],                                            │  │
│       │     data_start_row: 5,                                             │  │
│       │     headers: ["設備番号","設備名","点検項目","4月","5月",...] }      │  │
│       └──────────────────────────────────────────────────┘                  │
├─────────────────────────────────────────────────────────────────────────────┤
│ STEP 2: Excelコンテンツパターンの分類 (LLM)                                   │
│                                                                             │
│  3つの主要パターン:                                                           │
│                                                                             │
│  ┌─ パターンA: 機器+機器仕様のみ ─────────────────────────────────────┐     │
│  │ 設備番号 │ 設備名   │ メーカー │ 型式  │ 定格出力 │ 設置年 │       │     │
│  │ P-101   │ ポンプA  │ XXX社   │ AB-12 │ 15kW    │ 2015  │       │     │
│  │ → Assets + Specifications のみ生成                                │     │
│  │ → スキル: excel_import_asset_spec                                 │     │
│  └───────────────────────────────────────────────────────────────────┘     │
│                                                                             │
│  ┌─ パターンB: 機器+作業 (星取表形式) ────────────────────────────────┐     │
│  │ 設備番号 │ 点検項目   │ 4月 │ 5月 │ 6月 │ 7月 │ ...              │     │
│  │ P-101   │ 日常点検   │  ○  │  ○  │  △  │  ×  │ ...              │     │
│  │ P-101   │ 定期整備   │     │     │  ○  │     │ ...              │     │
│  │ → Assets + WorkOrders + WorkOrderLines (星取シンボル解析必要)      │     │
│  │ → スキル: excel_import_star_chart                                 │     │
│  └───────────────────────────────────────────────────────────────────┘     │
│                                                                             │
│  ┌─ パターンC: 機器仕様+作業 ─────────────────────────────────────────┐     │
│  │ 設備番号 │ メーカー │ 点検項目 │ 前回実施 │ 次回予定 │ 費用    │       │     │
│  │ P-101   │ XXX社   │ 定期点検 │ 2025/3  │ 2025/9  │ 50,000 │       │     │
│  │ → Assets + Specifications + WorkOrders + WorkOrderLines           │     │
│  │ → スキル: excel_import_full                                       │     │
│  └───────────────────────────────────────────────────────────────────┘     │
│                                                                             │
│  LLMによるパターン分類:                                                      │
│    プロンプト: "以下のヘッダーとサンプル5行から、データパターンを判定せよ"        │
│    → { pattern: "B", reason: "月名列あり、○×△記号あり → 星取表形式" }         │
├─────────────────────────────────────────────────────────────────────────────┤
│ STEP 3: 星取シンボル・凡例の解析 (LLM + ユーザー確認)                          │
│                                                                             │
│  パターンBの場合、○×△等のユーザー定義シンボルの意味を解決する必要がある         │
│                                                                             │
│  3a. Excel内の凡例検索:                                                      │
│    openpyxlで全シート・全セルをスキャン:                                       │
│      - "凡例", "記号", "マーク", "Legend" を含むセル周辺を探索                 │
│      - ヘッダー行の上のタイトル領域にある注記をチェック                         │
│      - 別シート「凡例」「マスタ」があるかチェック                               │
│    → 凡例発見例: "○:計画済, ×:未実施, △:一部実施, ●:完了"                     │
│                                                                             │
│  3b. LLMによる凡例推定 (凡例が見つからない場合):                               │
│    プロンプト: "星取表の以下のシンボルの意味を推定してください:                   │
│      使用されているシンボル: ○, ×, △, ●, ◎                                   │
│      コンテキスト: 保全計画表、月別の列"                                       │
│    → LLM推定: { "○":"計画あり", "●":"実施済", "△":"一部実施",                 │
│                  "×":"中止", "◎":"計画+実施済" }                              │
│                                                                             │
│  3c. ユーザーへの確認 (必須):                                                 │
│    ┌─── SSE → Frontend 確認ダイアログ ────────────────────────┐             │
│    │ 以下のシンボルマッピングでよろしいですか？                 │             │
│    │                                                          │             │
│    │  ○ → [計画済 (Planned=true)        ▼]                    │             │
│    │  ● → [実施済 (Actual=true)          ▼]                    │             │
│    │  △ → [計画済+一部実施               ▼]                    │             │
│    │  × → [中止 (Planned=false)          ▼]                    │             │
│    │  ◎ → [計画済+実施済                 ▼]                    │             │
│    │  (空白) → [データなし                ▼]                    │             │
│    │                                                          │             │
│    │  凡例出典: [Excel内凡例 / LLM推定]                        │             │
│    │                                                          │             │
│    │           [修正して適用]  [そのまま適用]                    │             │
│    └──────────────────────────────────────────────────────────┘             │
│                                                                             │
│  → ユーザー確認後、シンボルマッピングをStateに保持                              │
├─────────────────────────────────────────────────────────────────────────────┤
│ STEP 4: セル結合・複数行レイアウトの解析                                       │
│                                                                             │
│  問題: 1つの機器の情報が複数行にまたがる場合がある                              │
│                                                                             │
│  ケース1: セル結合 (merged_cells)                                             │
│    ┌──────────┬──────────┬────┬────┬────┐                                   │
│    │ P-101    │ 日常点検 │ ○  │ ○  │ ○  │                                   │
│    │ (結合)   ├──────────┼────┼────┼────┤                                   │
│    │          │ 定期整備 │    │ ○  │    │                                   │
│    │          ├──────────┼────┼────┼────┤                                   │
│    │          │ 振動測定 │ ○  │    │ ○  │                                   │
│    └──────────┴──────────┴────┴────┴────┘                                   │
│    → openpyxl merged_cells から結合範囲を検出                                 │
│    → 結合セルの値を結合範囲内の全行に展開 (unmerge + fill)                     │
│                                                                             │
│  ケース2: 値の空白継承 (結合なしだが値が省略されている)                          │
│    ┌──────────┬──────────┬────┬────┬────┐                                   │
│    │ P-101    │ 日常点検 │ ○  │ ○  │ ○  │                                   │
│    │          │ 定期整備 │    │ ○  │    │  ← 設備番号セルが空白               │
│    │          │ 振動測定 │ ○  │    │ ○  │  ← 前行の値を継承すべき             │
│    │ P-102    │ 日常点検 │ ○  │ ○  │ ○  │  ← 新しい設備                      │
│    └──────────┴──────────┴────┴────┴────┘                                   │
│    → LLMに先頭10行のパターンを見せて "値省略パターン" を検出                    │
│    → forward-fill ロジック: 空白セルに直上の非空白値を継承                      │
│    → 判定基準: 設備番号列で新値が出現するまで同一機器と判断                     │
│                                                                             │
│  ケース3: 複数行で1レコード (仕様が2行にまたがる等)                             │
│    → LLMがサンプルから行グルーピングルールを推定                                │
│    → ユーザー確認: "1機器あたり何行ですか？ [可変/固定N行]"                     │
│                                                                             │
│  処理フロー:                                                                  │
│    openpyxl読込み → 結合セル展開 → LLMレイアウト分析                           │
│    → 空白継承ルール適用 → 正規化された行列データに変換                          │
│    → この正規化データをPhase 2以降で使用                                       │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### Phase 2: マッピング生成 (パターン別LLM推論)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ THINKING PROCESS (LangGraph StateGraph)                                     │
│                                                                             │
│  State:                                                                     │
│  {                                                                          │
│    raw_data: 正規化済みExcelデータ (Phase1出力),                               │
│    header_row: 4,                                                           │
│    headers: ["設備番号","設備名","点検項目","4月","5月",...],                    │
│    pattern: "B" (星取表形式),                                                │
│    symbol_mapping: { "○":"planned", "●":"actual", ... },                    │
│    column_mapping: {},         ← LLMが生成                                   │
│    merge_rules: { ... },       ← Phase1で検出                                │
│    total_rows: 48000,                                                       │
│    processed_rows: 0,          ← チャンク処理の進捗                           │
│    chunk_size: 500,                                                         │
│    data_model: {},             ← 構築中のDataModel                           │
│    error_message: "",                                                       │
│    retry_count: 0                                                           │
│  }                                                                          │
│                                                                             │
│  ┌──────────────┐                                                           │
│  │ resolve_skill│ パターンに応じたスキルを解決:                                │
│  │              │   A → excel_import_asset_spec                              │
│  │              │   B → excel_import_star_chart                              │
│  │              │   C → excel_import_full                                    │
│  └──────┬───────┘                                                           │
│         ↓                                                                   │
│  ┌──────────────────┐                                                       │
│  │generate_operation│ LLMプロンプト (パターンB例):                             │
│  │                  │  System: "DataModel v3.0.0スキーマ + 星取表スキルctx"   │
│  │                  │  User: "ヘッダー: [設備番号,設備名,点検項目,4月,5月...]  │
│  │                  │   サンプル5行 + シンボルマッピング + 結合情報              │
│  │                  │   以下を判定せよ:                                       │
│  │                  │   1. どの列が Asset.id / Asset.name か                  │
│  │                  │   2. どの列が WorkOrder.name / Classification か        │
│  │                  │   3. どの列範囲が月別データ(WorkOrderLine)か             │
│  │                  │   4. 月列のフォーマット ('4月'→2025-04 変換ルール)        │
│  │                  │   5. 機器仕様に該当する列があるか"                       │
│  │                  │  → 出力:                                               │
│  │                  │   { column_mapping: {                                  │
│  │                  │       "設備番号": { target: "Asset.id", type: "key" },  │
│  │                  │       "設備名": { target: "Asset.name" },              │
│  │                  │       "点検項目": { target: "WorkOrder.name" },         │
│  │                  │       "4月": { target: "WorkOrderLine", month: 4 },    │
│  │                  │       "5月": { target: "WorkOrderLine", month: 5 },    │
│  │                  │       ...                                              │
│  │                  │     },                                                 │
│  │                  │     year_inference: 2025,  ← コンテキストから推定        │
│  │                  │     classification_guess: "05"                         │
│  │                  │   }                                                    │
│  └──────┬───────────┘                                                       │
│         ↓                                                                   │
│  ┌──────────────────┐                                                       │
│  │validate_operation│ マッピング整合性チェック:                                │
│  │                  │  - 必須: Asset.id列が特定されているか                    │
│  │                  │  - 月列が連続しているか                                  │
│  │                  │  - サンプル行で実際に変換テスト (5行)                     │
│  │                  │  - 変換テスト結果をユーザーに表示して確認依頼              │
│  └──────┬───────────┘                                                       │
│         ↓                                                                   │
│  ユーザー確認 (SSE → Frontend):                                              │
│    "以下のマッピングで取り込みます。よろしいですか？                              │
│     設備番号 → 機器ID, 点検項目 → 作業名, 4月〜3月 → 月別計画/実績             │
│     サンプル変換結果: P-101 ポンプA に 日常点検(計画)×12件 を生成               │

└─────────────────────────────────────────────────────────────────────────────┘
```

#### Phase 3: チャンク分割処理 (大容量Excel対応)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ 大容量対応設計 (5万行規模)                                                     │
│                                                                             │
│  問題:                                                                       │
│    - LLMコンテキストに全行は入らない                                           │
│    - 一括変換はメモリ・時間の制約がある                                         │
│    - 処理中断時に途中から再開できる必要がある                                    │
│                                                                             │
│  解決: ヘッダー条件保持 + チャンク分割処理 + 進捗管理                           │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────┐            │
│  │ Phase 1-2 で確定した情報 (LLM不要、決定論的に処理):          │            │
│  │   header_row: 4                                             │            │
│  │   column_mapping: { "設備番号"→Asset.id, ... }              │            │
│  │   symbol_mapping: { "○"→planned, ... }                      │            │
│  │   merge_rules: { forward_fill_columns: ["設備番号"] }        │            │
│  │   year_context: 2025                                        │            │
│  │   classification: "05"                                      │            │
│  └────────────────────────┬────────────────────────────────────┘            │
│                           │ (この情報をState固定値として保持)                  │
│                           ↓                                                 │
│  ┌─── チャンク処理ループ ──────────────────────────────────────┐            │
│  │                                                             │            │
│  │  total_rows = 48,000                                        │            │
│  │  chunk_size = 500 行                                        │            │
│  │  total_chunks = 96                                          │            │
│  │                                                             │            │
│  │  for chunk_idx in range(total_chunks):                      │            │
│  │    ┌─────────────────────────────────────────────────┐      │            │
│  │    │ 1. Excel読込み: rows[chunk_start : chunk_end]   │      │            │
│  │    │ 2. 結合セル展開 + 空白継承 (merge_rules適用)     │      │            │
│  │    │ 3. column_mapping に従いデータ変換:              │      │            │
│  │    │    - Asset 生成/更新 (重複IDは既存にマージ)       │      │            │
│  │    │    - WorkOrder 生成/更新                         │      │            │
│  │    │    - WorkOrderLine 生成                         │      │            │
│  │    │    - symbol_mapping でPlanned/Actual判定          │      │            │
│  │    │ 4. session.data_model にマージ                   │      │            │
│  │    │ 5. 進捗更新: processed_rows += chunk_size       │      │            │
│  │    └─────────────────────────────────────────────────┘      │            │
│  │    ↓                                                        │            │
│  │    SSE text_delta: "処理中... 2,500/48,000行 (5%)"          │            │
│  │                                                             │            │
│  │    ※ LLMは呼ばない (決定論的変換)                            │            │
│  │    ※ チャンク間でAsset/WOの重複を統合管理                     │            │
│  │                                                             │            │
│  └─────────────────────────────────────────────────────────────┘            │
│                                                                             │
│  進捗管理 (中断・再開対応):                                                    │
│  ┌─────────────────────────────────────────────────────────────┐            │
│  │ session.import_state = {                                     │            │
│  │   file_hash: "abc123",      ← ファイル同一性確認              │            │
```
