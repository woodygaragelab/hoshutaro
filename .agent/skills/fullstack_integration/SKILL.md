# Full-Stack Integration Skill

このスキルは、フロントエンドのコンポーネント改修やバックエンド（API）の処理追加を行った後、相互の連携（データのやり取りやUIへの反映）が途切れたり、繋ぎ込みを忘れたりするミスを防止するためのチェック手順を定義します。

## 1. 原則（The Golden Rule）

**「単体テストで終わらせない。必ずフロントエンドのUI操作からバックエンドの処理を経て、最終的にUIへ結果が返る一連のループ（End-to-End）を繋ぐこと」**

フロント側での「ボタン追加」や、バックエンド側での「ロジック変更」など、片側だけの作業を行った場合でも、必ずもう片方との接続部分（API呼び出し、Propsの伝播）をチェックしなければなりません。

## 2. 実装および繋ぎ込み時の必須チェック項目

機能を実装した後は、以下の配線（Wiring）がすべて完了しているか、ソースコード上で確認してください。

### [ ] 2-1. API通信インターフェースの一致
- フロントエンドの `fetch` や `axios` で指定しているURL（エンドポイント）とHTTPメソッドが、バックエンド（FastAPI等）のルーター定義と完全に一致しているか。
- 送信するペイロードのJSON構造が、バックエンド側が期待するスキーマ（Pydanticモデル等）と一致しているか。

### [ ] 2-2. フロントエンド内の状態管理とPropsの伝播
- 画面上の「ボタン」や「フォーム」のアクション（`onClick` や `onSubmit`）から、実際にAPIを呼び出す関数までのパスが繋がっているか。（空の関数や `console.log` だけで止まっていないか）
- APIから返ってきたレスポンスデータを、Reactの `useState` や管理コンポーネントに正しく格納し、再レンダリングをトリガーできているか。

### [ ] 2-3. UIへのフィードバックの実装
- 通信中であることを示すローディング表示が正しく動作・解除されるか。
- バックエンドからエラーが返ってきた場合、ただ例外を投げるだけでなく、ユーザー画面にエラーメッセージ（トーストやSnackbarなど）として表示できているか。

## 3. End-to-End (E2E) 検証プロセス

コードレベルの配線チェックが終わった後は、先述の `Browser UI Testing Skill` と連携し、以下のプロセスで結合テストを行います：

1. **バックエンド・フロントエンドの稼働確認**: 統合前に両方のサーバーがエラーなく動いているか確認する。
2. **AI自身によるE2Eリクエスト**: ターミナルなどからcurlコマンドでバックエンドAPIを直接叩き、データが想定通りに返ってくることを確認。
3. **ブラウザ経由の導通確認**: `browser_subagent` を使ってフロントエンドのUIを操作し、ネットワークエラー（CORSや404, 422 Unprocessable Entityなど）が起きずに処理が完了するかを確実にテストする。

## 4. エラー時の自己修正

未配線や接続エラー（型不一致など）を発見した場合、一時的なモックデータに切り替えて誤魔化すのではなく、「なぜ繋がらなかったのか」を特定し、必ずインテグレーションを完了させること。
